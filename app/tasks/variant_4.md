## 1. Орташадан үлкен ең кіші жоқ оң сан (Оңай)

> **Русский:** Наименьшее отсутствующее положительное, большее среднего (Лёгкий)

### Шарт (Қазақша)

`nums` бүтін сандар массиві берілген.

`nums`-те **жоқ** және `nums`-тің барлық элементтерінің **орташасынан қатаң үлкен** ең кіші оң бүтін санды қайтарыңыз.

> Массивтің орташасы — барлық элементтер қосындысының элементтер санына қатынасы.

**Мысалдар:**

**1-мысал:**
- Кіріс: `nums = [3,5]`
- Шығыс: `6`
- Түсінік: Орташа = `(3+5)/2 = 4`. 4-тен үлкен және массивте жоқ ең кіші оң сан — **6**.

**2-мысал:**
- Кіріс: `nums = [-1,1,2]`
- Шығыс: `3`
- Түсінік: Орташа = `(-1+1+2)/3 ≈ 0.667`. 0.667-ден үлкен және массивте жоқ ең кіші оң сан — **3** (`1` және `2` массивте бар).

**3-мысал:**
- Кіріс: `nums = [4,-1]`
- Шығыс: `2`
- Түсінік: Орташа = `(4+(-1))/2 = 1.5`. 1.5-тен үлкен және массивте жоқ ең кіші оң сан — **2**.

**Шектеулер:**
- `1 <= nums.length <= 100`
- `-100 <= nums[i] <= 100`

### Условие (Русский)

Дан массив целых чисел `nums`.

Верните наименьшее положительное целое число, которого **нет** в `nums` и которое **строго больше** среднего значения всех элементов `nums`.

> Среднее массива — сумма всех элементов, делённая на их количество.

**Примеры:**

**Пример 1:**
- Ввод: `nums = [3,5]`
- Вывод: `6`
- Пояснение: Среднее = `(3+5)/2 = 4`. Наименьшее отсутствующее положительное, большее 4 — **6**.

**Пример 2:**
- Ввод: `nums = [-1,1,2]`
- Вывод: `3`
- Пояснение: Среднее = `(-1+1+2)/3 ≈ 0.667`. Наименьшее отсутствующее положительное, большее 0.667 — **3** (числа `1` и `2` уже есть в массиве).

**Пример 3:**
- Ввод: `nums = [4,-1]`
- Вывод: `2`
- Пояснение: Среднее = `(4+(-1))/2 = 1.5`. Наименьшее отсутствующее положительное, большее 1.5 — **2**.

**Ограничения:**
- `1 <= nums.length <= 100`
- `-100 <= nums[i] <= 100`

## 2. Үй тонаушы V (Орташа)

> **Русский:** Грабитель домов V (Средний)

### Шарт (Қазақша)

Сіз көшедегі үйлерді тонауды жоспарлаған кәсіби тонаушысыз. Әр үйде белгілі мөлшерде ақша бар және ол түс кодымен қорғалған қауіпсіздік жүйесімен жабдықталған.

`n` ұзындықты екі бүтін сандар массиві `nums` және `colors` берілген, мұнда `nums[i]` — `i`-ші үйдегі ақша мөлшері, `colors[i]` — сол үйдің түс коды.

**Бір түс кодын бөлісетін екі іргелес үйді тонауға болмайды.**

Тоналуы мүмкін максималды ақша мөлшерін қайтарыңыз.

**Мысалдар:**

**1-мысал:**
- Кіріс: `nums = [1,4,3,5], colors = [1,1,2,2]`
- Шығыс: `9`
- Түсінік: `i=1` (`nums[1]=4`) және `i=3` (`nums[3]=5`) үйлерін таңдаймыз — олар іргелес емес. Жалпы: `4+5=9`.

**2-мысал:**
- Кіріс: `nums = [3,1,2,4], colors = [2,3,2,2]`
- Шығыс: `8`
- Түсінік: `i=0` (`3`), `i=1` (`1`), `i=3` (`4`) үйлерін таңдаймыз. `i=0` мен `i=1` — түрлі түстер, `i=3` — `i=1`-мен іргелес емес. Жалпы: `3+1+4=8`.

**3-мысал:**
- Кіріс: `nums = [10,1,3,9], colors = [1,1,1,2]`
- Шығыс: `22`
- Түсінік: `i=0` (`10`), `i=2` (`3`), `i=3` (`9`) үйлерін таңдаймыз. `i=0` мен `i=2` — іргелес емес, `i=2` мен `i=3` — түрлі түстер. Жалпы: `10+3+9=22`.

**Шектеулер:**
- `1 <= n == nums.length == colors.length <= 10^5`
- `1 <= nums[i], colors[i] <= 10^5`

### Условие (Русский)

Вы профессиональный грабитель, планирующий ограбить дома вдоль улицы. В каждом доме хранится определённая сумма денег, и он защищён системой безопасности с цветовым кодом.

Даны два целочисленных массива `nums` и `colors` длины `n`, где `nums[i]` — сумма денег в `i`-м доме, `colors[i]` — цветовой код этого дома.

**Нельзя грабить два соседних дома, если у них одинаковый цветовой код.**

Верните максимальную сумму, которую можно ограбить.

**Примеры:**

**Пример 1:**
- Ввод: `nums = [1,4,3,5], colors = [1,1,2,2]`
- Вывод: `9`
- Пояснение: Выбираем `i=1` (`nums[1]=4`) и `i=3` (`nums[3]=5`) — они не соседние. Итого: `4+5=9`.

**Пример 2:**
- Ввод: `nums = [3,1,2,4], colors = [2,3,2,2]`
- Вывод: `8`
- Пояснение: Выбираем `i=0` (`3`), `i=1` (`1`), `i=3` (`4`). `i=0` и `i=1` — разные цвета, `i=3` не соседний с `i=1`. Итого: `3+1+4=8`.

**Пример 3:**
- Ввод: `nums = [10,1,3,9], colors = [1,1,1,2]`
- Вывод: `22`
- Пояснение: Выбираем `i=0` (`10`), `i=2` (`3`), `i=3` (`9`). `i=0` и `i=2` — не соседние, `i=2` и `i=3` — разные цвета. Итого: `10+3+9=22`.

**Ограничения:**
- `1 <= n == nums.length == colors.length <= 10^5`
- `1 <= nums[i], colors[i] <= 10^5`

## 3. Бөлінетін цифр көбейтіндісі бар ең кіші нөлсіз сан II (Қиын)

> **Русский:** Наименьшее беснулевое число с делимым произведением цифр II (Сложный)

### Шарт (Қазақша)

Оң бүтін санды білдіретін `num` жолы және `t` бүтін саны берілген.

Сан **нөлсіз** деп аталады, егер оның цифрларының ешқайсысы `0` болмаса.

`num`-нен **үлкен немесе тең** және цифрларының **көбейтіндісі `t`-ге бөлінетін** ең кіші нөлсіз санды жол ретінде қайтарыңыз. Мұндай сан болмаса, `"-1"` қайтарыңыз.

**Мысалдар:**

**1-мысал:**
- Кіріс: `num = "1234", t = 256`
- Шығыс: `"1488"`
- Түсінік: 1234-тен үлкен, нөлсіз, цифрлары көбейтіндісі 256-ға бөлінетін ең кіші сан — **1488** (көбейтінді = `1×4×8×8 = 256`).

**2-мысал:**
- Кіріс: `num = "12355", t = 50`
- Шығыс: `"12355"`
- Түсінік: 12355 — нөлсіз, цифрлары көбейтіндісі `1×2×3×5×5 = 150`, ал 150 — 50-ге бөлінеді. Жауап өзі.

**3-мысал:**
- Кіріс: `num = "11111", t = 26`
- Шығыс: `"-1"`
- Түсінік: 11111-ден үлкен және цифрлары көбейтіндісі 26-ға бөлінетін сан жоқ (26 = 2 × 13, ал 13 — цифр емес).

**Шектеулер:**
- `2 <= num.length <= 2 × 10^5`
- `num` тек `['0', '9']` аралығындағы цифрлардан тұрады.
- `num`-де жетекші нөлдер жоқ.
- `1 <= t <= 10^14`

### Условие (Русский)

Дана строка `num`, представляющая положительное целое число, и целое число `t`.

Число называется **беснулевым**, если ни одна из его цифр не равна `0`.

Верните строку, представляющую наименьшее беснулевое число, **большее или равное `num`**, произведение цифр которого **делится на `t`**. Если такого числа не существует, верните `"-1"`.

**Примеры:**

**Пример 1:**
- Ввод: `num = "1234", t = 256`
- Вывод: `"1488"`
- Пояснение: Наименьшее беснулевое число ≥ 1234 с произведением цифр, делящимся на 256 — это **1488** (произведение = `1×4×8×8 = 256`).

**Пример 2:**
- Ввод: `num = "12355", t = 50`
- Вывод: `"12355"`
- Пояснение: 12355 уже является беснулевым числом, произведение цифр `1×2×3×5×5 = 150` делится на 50.

**Пример 3:**
- Ввод: `num = "11111", t = 26`
- Вывод: `"-1"`
- Пояснение: Не существует числа ≥ 11111 с произведением цифр, делящимся на 26 (26 = 2 × 13, а 13 не является цифрой).

**Ограничения:**
- `2 <= num.length <= 2 × 10^5`
- `num` состоит только из цифр в диапазоне `['0', '9']`.
- `num` не содержит ведущих нулей.
- `1 <= t <= 10^14`
