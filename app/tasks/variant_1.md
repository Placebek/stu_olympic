## 1. Тең көбейтінділі максималды ішкі массив (Оңай)

> **Русский:** Максимальный подмассив с равными произведениями (Лёгкий)

### Шарт (Қазақша)

`nums` — оң бүтін сандардан тұратын массив берілген.

`arr` массиві **көбейтіндісі тең** деп аталады, егер мына шарт орындалса:

```
prod(arr) == lcm(arr) * gcd(arr)
```

мұнда:
- `prod(arr)` — массивтің барлық элементтерінің көбейтіндісі,
- `gcd(arr)` — массивтің барлық элементтерінің ең үлкен ортақ бөлгіші,
- `lcm(arr)` — массивтің барлық элементтерінің ең кіші ортақ еселігі.

`nums` массивінің ең ұзын көбейтіндісі тең ішкі массивінің ұзындығын қайтарыңыз.

**Мысалдар:**

**1-мысал:**
- Кіріс: `nums = [1,2,1,2,1,1,1]`
- Шығыс: `5`
- Түсінік: Ең ұзын ішкі массив — `[1, 2, 1, 1, 1]`, мұнда `prod = 2`, `gcd = 1`, `lcm = 2`.

**2-мысал:**
- Кіріс: `nums = [2,3,4,5,6]`
- Шығыс: `3`
- Түсінік: Ең ұзын ішкі массив — `[3, 4, 5]`.

**3-мысал:**
- Кіріс: `nums = [1,2,3,1,4,5,1]`
- Шығыс: `5`

**Шектеулер:**
- `2 <= nums.length <= 100`
- `1 <= nums[i] <= 10`

---

### Условие (Русский)

Дан массив положительных целых чисел `nums`.

Массив `arr` называется **эквивалентным по произведению**, если выполняется условие:

```
prod(arr) == lcm(arr) * gcd(arr)
```

где:
- `prod(arr)` — произведение всех элементов массива,
- `gcd(arr)` — наибольший общий делитель всех элементов массива,
- `lcm(arr)` — наименьшее общее кратное всех элементов массива.

Верните длину наидлиннейшего эквивалентного по произведению подмассива массива `nums`.

**Примеры:**

**Пример 1:**
- Ввод: `nums = [1,2,1,2,1,1,1]`
- Вывод: `5`
- Пояснение: Наидлиннейший эквивалентный подмассив — `[1, 2, 1, 1, 1]`, где `prod = 2`, `gcd = 1`, `lcm = 2`.

**Пример 2:**
- Ввод: `nums = [2,3,4,5,6]`
- Вывод: `3`
- Пояснение: Наидлиннейший эквивалентный подмассив — `[3, 4, 5]`.

**Пример 3:**
- Ввод: `nums = [1,2,3,1,4,5,1]`
- Вывод: `5`

**Ограничения:**
- `2 <= nums.length <= 100`
- `1 <= nums[i] <= 10`

## 2. Қажетті заттарды сатып алудың ең аз құны (Орташа)

> **Русский:** Минимальная стоимость приобретения необходимых предметов (Средний)

### Шарт (Қазақша)

Бес бүтін сан берілген: `cost1`, `cost2`, `costBoth`, `need1` және `need2`.

Үш түрлі зат қол жетімді:
- **1-түрдегі зат** `cost1` тұрады және тек 1-түр талабына 1 бірлік береді.
- **2-түрдегі зат** `cost2` тұрады және тек 2-түр талабына 1 бірлік береді.
- **3-түрдегі зат** `costBoth` тұрады және 1-түр мен 2-түр талаптарының екеуіне де бір мезгілде 1 бірлік береді.

1-түрге жалпы үлес кемінде `need1`, ал 2-түрге кемінде `need2` болатындай жеткілікті зат жинау керек.

Осы талаптарды орындаудың ең аз жалпы құнын бүтін сан ретінде қайтарыңыз.

**Мысалдар:**

**1-мысал:**
- Кіріс: `cost1=3, cost2=2, costBoth=1, need1=3, need2=2`
- Шығыс: `3`
- Түсінік: 3-түрден 3 зат сатып аламыз: `3×1=3`. 1-түрге үлес: 3 ≥ need1, 2-түрге: 3 ≥ need2.

**2-мысал:**
- Кіріс: `cost1=5, cost2=4, costBoth=15, need1=2, need2=3`
- Шығыс: `22`
- Түсінік: 2 дана 1-түр және 3 дана 2-түр сатып аламыз: `2×5 + 3×4 = 22`.

**3-мысал:**
- Кіріс: `cost1=5, cost2=4, costBoth=15, need1=0, need2=0`
- Шығыс: `0`
- Түсінік: Талаптар нөлдік — ештеңе сатып алу қажет емес.

**Шектеулер:**
- `1 <= cost1, cost2, costBoth <= 10^6`
- `0 <= need1, need2 <= 10^9`

---

### Условие (Русский)

Даны пять целых чисел: `cost1`, `cost2`, `costBoth`, `need1` и `need2`.

Доступны три типа предметов:
- **Предмет типа 1** стоит `cost1` и даёт 1 единицу только к требованию типа 1.
- **Предмет типа 2** стоит `cost2` и даёт 1 единицу только к требованию типа 2.
- **Предмет типа 3** стоит `costBoth` и даёт 1 единицу одновременно к требованиям типа 1 и типа 2.

Необходимо набрать достаточное количество предметов, чтобы суммарный вклад в тип 1 был не менее `need1`, а в тип 2 — не менее `need2`.

Верните целое число — минимально возможную суммарную стоимость для выполнения этих требований.

**Примеры:**

**Пример 1:**
- Ввод: `cost1=3, cost2=2, costBoth=1, need1=3, need2=2`
- Вывод: `3`
- Пояснение: Покупаем 3 предмета типа 3 за `3×1=3`. Вклад в тип 1: 3 ≥ need1, в тип 2: 3 ≥ need2.

**Пример 2:**
- Ввод: `cost1=5, cost2=4, costBoth=15, need1=2, need2=3`
- Вывод: `22`
- Пояснение: Покупаем 2 предмета типа 1 и 3 предмета типа 2: `2×5 + 3×4 = 22`.

**Пример 3:**
- Ввод: `cost1=5, cost2=4, costBoth=15, need1=0, need2=0`
- Вывод: `0`
- Пояснение: Требования нулевые — ничего покупать не нужно.

**Ограничения:**
- `1 <= cost1, cost2, costBoth <= 10^6`
- `0 <= need1, need2 <= 10^9`

## 3. Барлық адамдарды өткізудің ең аз уақыты (Қиын)

> **Русский:** Минимальное время для переправки всех людей (Сложный)

### Шарт (Қазақша)

Базалық лагерьде `n` адам бар, олар қайықпен өзенді кесіп өтіп, межелі жерге жетуі керек. Қайыққа бір жолы ең көп `k` адам сияды. Өту жағдайы `m` сатыдан тұратын циклдік ортаға байланысты.

Әр `j` сатысының жылдамдық коэффициенті `mul[j]` болады:
- `mul[j] > 1` болса — өту баяулайды.
- `mul[j] < 1` болса — өту жылдамдайды.

Әр `i` адамының ескек ерекшелігі `time[i]` — бейтарап жағдайда жалғыз өту уақыты (минутпен).

**Ережелер:**
- `j` сатысында жолға шыққан `g` тобы, мүшелерінің `time[i]` максимумын `mul[j]`-ға көбейткен уақыт жұмсайды.
- Топ `d` уақытта өткеннен кейін, саты `floor(d) % m` қадамға жылжиды.
- Жағада адам қалса, біреу қайықпен қайтуы керек. `r` адамының қайтуы `time[r] × mul[ағымдағы_саты]` минут алады; саты дәл сол сияқты жылжиды.

Барлық адамдарды өткізудің ең аз жалпы уақытын қайтарыңыз. Егер бұл мүмкін болмаса — `-1` қайтарыңыз.

**Мысалдар:**

**1-мысал:**
- Кіріс: `n=1, k=1, m=2, time=[5], mul=[1.0,1.3]`
- Шығыс: `5.00000`
- Түсінік: 0-адам 0-сатыдан жолға шығады: `5×1.00=5.00` минут. Барлығы өтті.

**2-мысал:**
- Кіріс: `n=3, k=2, m=3, time=[2,5,8], mul=[1.0,1.5,0.75]`
- Шығыс: `14.50000`
- Түсінік:
  1. 0 және 2 адамдар өтеді: `max(2,8)×1.00=8.00` мин. Саты: `(0+floor(8)%3)%3=2`.
  2. 0-адам қайтады: `2×0.75=1.50` мин. Саты: `(2+floor(1.5)%3)%3=0`.
  3. 0 және 1 адамдар өтеді: `max(2,5)×1.00=5.00` мин.
  4. Барлығы: `8.00+1.50+5.00=14.50`.

**3-мысал:**
- Кіріс: `n=2, k=1, m=2, time=[10,10], mul=[2.0,2.0]`
- Шығыс: `-1.00000`
- Түсінік: Қайыққа бір адам сияды, екіншісі қайтуы керек — өту мүмкін емес.

**Шектеулер:**
- `1 <= n == time.length <= 12`
- `1 <= k <= 5`
- `1 <= m <= 5`
- `1 <= time[i] <= 100`
- `m == mul.length`
- `0.5 <= mul[i] <= 2.0`

---

### Условие (Русский)

Дано `n` человек в базовом лагере, которым нужно переправиться через реку на лодке к месту назначения. Лодка может вмещать не более `k` человек за раз. Переправа зависит от условий среды, которые циклически меняются по `m` стадиям.

Каждая стадия `j` имеет множитель скорости `mul[j]`:
- Если `mul[j] > 1` — переправа замедляется.
- Если `mul[j] < 1` — переправа ускоряется.

Каждый человек `i` имеет характеристику гребли `time[i]` — время (в минутах) его одиночной переправы в нейтральных условиях.

**Правила:**
- Группа `g`, отплывающая на стадии `j`, затрачивает время, равное максимальному `time[i]` среди её участников, умноженному на `mul[j]`.
- После переправы группы за время `d` стадия сдвигается на `floor(d) % m` шагов вперёд.
- Если на берегу остались люди, один человек должен вернуться с лодкой. Возвращение человека `r` занимает `time[r] × mul[текущая_стадия]` минут; стадия сдвигается аналогично.

Верните минимальное суммарное время для переправки всех людей. Если это невозможно — верните `-1`.

**Примеры:**

**Пример 1:**
- Ввод: `n=1, k=1, m=2, time=[5], mul=[1.0,1.3]`
- Вывод: `5.00000`
- Пояснение: Человек 0 отплывает на стадии 0: `5×1.00=5.00` минут. Все переправились.

**Пример 2:**
- Ввод: `n=3, k=2, m=3, time=[2,5,8], mul=[1.0,1.5,0.75]`
- Вывод: `14.50000`
- Пояснение:
  1. Люди 0 и 2 переправляются: `max(2,8)×1.00=8.00` мин. Стадия: `(0+floor(8)%3)%3=2`.
  2. Человек 0 возвращается: `2×0.75=1.50` мин. Стадия: `(2+floor(1.5)%3)%3=0`.
  3. Люди 0 и 1 переправляются: `max(2,5)×1.00=5.00` мин.
  4. Итого: `8.00+1.50+5.00=14.50`.

**Пример 3:**
- Ввод: `n=2, k=1, m=2, time=[10,10], mul=[2.0,2.0]`
- Вывод: `-1.00000`
- Пояснение: Лодка вмещает одного, второй должен вернуться — переправа невозможна.

**Ограничения:**
- `1 <= n == time.length <= 12`
- `1 <= k <= 5`
- `1 <= m <= 5`
- `1 <= time[i] <= 100`
- `m == mul.length`
- `0.5 <= mul[i] <= 2.0`
