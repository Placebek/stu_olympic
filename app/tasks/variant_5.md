## 1. Әдемі жұптар саны (Оңай)

> **Русский:** Количество красивых пар (Лёгкий)

### Шарт (Қазақша)

0-индекстелген `nums` бүтін сандар массиві берілген.

`0 <= i < j < nums.length` шартын қанағаттандыратын `i`, `j` индекстер жұбы **әдемі** деп аталады, егер `nums[i]`-дің **бірінші цифры** мен `nums[j]`-дің **соңғы цифры** өзара жай болса.

`nums`-тегі әдемі жұптардың жалпы санын қайтарыңыз.

> Екі `x` және `y` бүтін сан **өзара жай** деп аталады, егер екеуін де бөлетін 1-ден үлкен бүтін сан болмаса, яғни `gcd(x, y) == 1`.

**Мысалдар:**

**1-мысал:**
- Кіріс: `nums = [2,5,1,4]`
- Шығыс: `5`
- Түсінік: 5 әдемі жұп бар:
  - `i=0, j=1`: бірінші цифр `2`, соңғы цифр `5` → `gcd(2,5)=1` ✓
  - `i=0, j=2`: бірінші цифр `2`, соңғы цифр `1` → `gcd(2,1)=1` ✓
  - `i=1, j=2`: бірінші цифр `5`, соңғы цифр `1` → `gcd(5,1)=1` ✓
  - `i=1, j=3`: бірінші цифр `5`, соңғы цифр `4` → `gcd(5,4)=1` ✓
  - `i=2, j=3`: бірінші цифр `1`, соңғы цифр `4` → `gcd(1,4)=1` ✓

**2-мысал:**
- Кіріс: `nums = [11,21,12]`
- Шығыс: `2`
- Түсінік:
  - `i=0, j=1`: бірінші цифр `1`, соңғы цифр `1` → `gcd(1,1)=1` ✓
  - `i=0, j=2`: бірінші цифр `1`, соңғы цифр `2` → `gcd(1,2)=1` ✓

**Шектеулер:**
- `2 <= nums.length <= 100`
- `1 <= nums[i] <= 9999`
- `nums[i] % 10 != 0`

---

### Условие (Русский)

Дан 0-индексированный массив целых чисел `nums`.

Пара индексов `i`, `j`, где `0 <= i < j < nums.length`, называется **красивой**, если **первая цифра** `nums[i]` и **последняя цифра** `nums[j]` взаимно просты.

Верните общее количество красивых пар в `nums`.

> Два целых числа `x` и `y` **взаимно просты**, если нет целого числа больше 1, делящего оба, то есть `gcd(x, y) == 1`.

**Примеры:**

**Пример 1:**
- Ввод: `nums = [2,5,1,4]`
- Вывод: `5`
- Пояснение: 5 красивых пар:
  - `i=0, j=1`: первая цифра `2`, последняя `5` → `gcd(2,5)=1` ✓
  - `i=0, j=2`: первая цифра `2`, последняя `1` → `gcd(2,1)=1` ✓
  - `i=1, j=2`: первая цифра `5`, последняя `1` → `gcd(5,1)=1` ✓
  - `i=1, j=3`: первая цифра `5`, последняя `4` → `gcd(5,4)=1` ✓
  - `i=2, j=3`: первая цифра `1`, последняя `4` → `gcd(1,4)=1` ✓

**Пример 2:**
- Ввод: `nums = [11,21,12]`
- Вывод: `2`
- Пояснение:
  - `i=0, j=1`: первая цифра `1`, последняя `1` → `gcd(1,1)=1` ✓
  - `i=0, j=2`: первая цифра `1`, последняя `2` → `gcd(1,2)=1` ✓

**Ограничения:**
- `2 <= nums.length <= 100`
- `1 <= nums[i] <= 9999`
- `nums[i] % 10 != 0`

## 2. Үй бояу IV (Орташа)

> **Русский:** Покраска домов IV (Средний)

### Шарт (Қазақша)

Жол бойында орналасқан үйлер санын білдіретін жұп бүтін сан `n` және `n x 3` өлшемді `cost` екіөлшемді массиві берілген, мұнда `cost[i][j]` — `i`-ші үйді `j+1` түспен бояудың құны.

Үйлер **әдемі** болып саналады, егер мына шарттар орындалса:
- Екі іргелес үйдің түсі бірдей болмауы керек.
- Қатардың екі шетінен бірдей қашықтықтағы үйлердің түсі бірдей болмауы керек. Мысалы, `n=6` болса, `(0,5)`, `(1,4)`, `(2,3)` позициялардағы үйлер бірдей қашықтықта.

Үйлер **әдемі** болатындай бояудың **ең аз құнын** қайтарыңыз.

**Мысалдар:**

**1-мысал:**
- Кіріс: `n = 4, cost = [[3,5,7],[6,2,9],[4,8,1],[7,3,5]]`
- Шығыс: `9`
- Түсінік: Оңтайлы бояу реті `[1,2,3,2]`, шығындар `[3,2,1,3]`:
  - Іргелес үйлердің түсі бірдей емес ✓
  - `0` және `3` позициялары: `1 ≠ 2` ✓
  - `1` және `2` позициялары: `2 ≠ 3` ✓
  - Жалпы: `3+2+1+3=9`

**2-мысал:**
- Кіріс: `n = 6, cost = [[2,4,6],[5,3,8],[7,1,9],[4,6,2],[3,5,7],[8,2,4]]`
- Шығыс: `18`
- Түсінік: Оңтайлы бояу реті `[1,3,2,3,1,2]`, шығындар `[2,8,1,2,3,2]`:
  - `0` және `5`: `1 ≠ 2` ✓
  - `1` және `4`: `3 ≠ 1` ✓
  - `2` және `3`: `2 ≠ 3` ✓
  - Жалпы: `2+8+1+2+3+2=18`

**Шектеулер:**
- `2 <= n <= 10^5`
- `n` — жұп сан.
- `cost.length == n`
- `cost[i].length == 3`
- `0 <= cost[i][j] <= 10^5`

---

### Условие (Русский)

Дано чётное целое число `n` — количество домов, расположенных в ряд, и двумерный массив `cost` размера `n x 3`, где `cost[i][j]` — стоимость покраски `i`-го дома в цвет `j+1`.

Дома выглядят **красиво**, если выполнены следующие условия:
- Никакие два соседних дома не покрашены в одинаковый цвет.
- Дома, равноудалённые от концов ряда, не покрашены в одинаковый цвет. Например, при `n=6` равноудалёнными являются пары `(0,5)`, `(1,4)`, `(2,3)`.

Верните минимальную стоимость покраски домов так, чтобы они выглядели **красиво**.

**Примеры:**

**Пример 1:**
- Ввод: `n = 4, cost = [[3,5,7],[6,2,9],[4,8,1],[7,3,5]]`
- Вывод: `9`
- Пояснение: Оптимальная последовательность `[1,2,3,2]`, затраты `[3,2,1,3]`:
  - Соседние дома разных цветов ✓
  - Позиции `0` и `3`: `1 ≠ 2` ✓
  - Позиции `1` и `2`: `2 ≠ 3` ✓
  - Итого: `3+2+1+3=9`

**Пример 2:**
- Ввод: `n = 6, cost = [[2,4,6],[5,3,8],[7,1,9],[4,6,2],[3,5,7],[8,2,4]]`
- Вывод: `18`
- Пояснение: Оптимальная последовательность `[1,3,2,3,1,2]`, затраты `[2,8,1,2,3,2]`:
  - Позиции `0` и `5`: `1 ≠ 2` ✓
  - Позиции `1` и `4`: `3 ≠ 1` ✓
  - Позиции `2` и `3`: `2 ≠ 3` ✓
  - Итого: `2+8+1+2+3+2=18`

**Ограничения:**
- `2 <= n <= 10^5`
- `n` — чётное число.
- `cost.length == n`
- `cost[i].length == 3`
- `0 <= cost[i][j] <= 10^5`

## 3. Тиімді ішкі тізбектер саны (Қиын)

> **Русский:** Количество эффективных подпоследовательностей (Сложный)

### Шарт (Қазақша)

`nums` бүтін сандар массиві берілген.

Массивтің **күші** — оның барлық элементтерінің побиттік OR-ы ретінде анықталады.

Ішкі тізбек **тиімді** деп аталады, егер оны жою қалған элементтердің күшін **қатаң азайтса**.

`nums`-тегі тиімді ішкі тізбектердің санын қайтарыңыз. Жауап өте үлкен болуы мүмкін, сондықтан оны `10^9 + 7` модулі бойынша қайтарыңыз.

> Бос массивтің побиттік OR-ы `0`-ге тең.

**Мысалдар:**

**1-мысал:**
- Кіріс: `nums = [1,2,3]`
- Шығыс: `3`
- Түсінік: Массивтің күші = `1 OR 2 OR 3 = 3`. Тиімді ішкі тізбектер:
  - `[1,3]`: қалған `[2]` → OR = `2` < `3` ✓
  - `[2,3]`: қалған `[1]` → OR = `1` < `3` ✓
  - `[1,2,3]`: қалған `[]` → OR = `0` < `3` ✓

**2-мысал:**
- Кіріс: `nums = [7,4,6]`
- Шығыс: `4`
- Түсінік: Массивтің күші = `7 OR 4 OR 6 = 7`. Тиімді ішкі тізбектер:
  - `[7]`: қалған `[4,6]` → OR = `6` < `7` ✓
  - `[7,4]`: қалған `[6]` → OR = `6` < `7` ✓
  - `[7,6]`: қалған `[4]` → OR = `4` < `7` ✓
  - `[7,4,6]`: қалған `[]` → OR = `0` < `7` ✓

**3-мысал:**
- Кіріс: `nums = [8,8]`
- Шығыс: `1`
- Түсінік: Массивтің күші = `8`. Тек `[8,8]` тиімді — жойғаннан кейін OR = `0` < `8` ✓.

**4-мысал:**
- Кіріс: `nums = [2,2,1]`
- Шығыс: `5`
- Түсінік: Массивтің күші = `2 OR 2 OR 1 = 3`. Тиімді ішкі тізбектер:
  - `[1]`: қалған `[2,2]` → OR = `2` < `3` ✓
  - `[2,1]` (`nums[0], nums[2]`): қалған `[2]` → OR = `2` < `3` ✓
  - `[2,1]` (`nums[1], nums[2]`): қалған `[2]` → OR = `2` < `3` ✓
  - `[2,2]`: қалған `[1]` → OR = `1` < `3` ✓
  - `[2,2,1]`: қалған `[]` → OR = `0` < `3` ✓

**Шектеулер:**
- `1 <= nums.length <= 10^5`
- `1 <= nums[i] <= 10^6`

---

### Условие (Русский)

Дан массив целых чисел `nums`.

**Сила** массива определяется как побитовое OR всех его элементов.

Подпоследовательность считается **эффективной**, если её удаление **строго уменьшает** силу оставшихся элементов.

Верните количество эффективных подпоследовательностей в `nums`. Так как ответ может быть большим, верните его по модулю `10^9 + 7`.

> Побитовое OR пустого массива равно `0`.

**Примеры:**

**Пример 1:**
- Ввод: `nums = [1,2,3]`
- Вывод: `3`
- Пояснение: Сила массива = `1 OR 2 OR 3 = 3`. Эффективные подпоследовательности:
  - `[1,3]`: остаток `[2]` → OR = `2` < `3` ✓
  - `[2,3]`: остаток `[1]` → OR = `1` < `3` ✓
  - `[1,2,3]`: остаток `[]` → OR = `0` < `3` ✓

**Пример 2:**
- Ввод: `nums = [7,4,6]`
- Вывод: `4`
- Пояснение: Сила массива = `7 OR 4 OR 6 = 7`. Эффективные подпоследовательности:
  - `[7]`: остаток `[4,6]` → OR = `6` < `7` ✓
  - `[7,4]`: остаток `[6]` → OR = `6` < `7` ✓
  - `[7,6]`: остаток `[4]` → OR = `4` < `7` ✓
  - `[7,4,6]`: остаток `[]` → OR = `0` < `7` ✓

**Пример 3:**
- Ввод: `nums = [8,8]`
- Вывод: `1`
- Пояснение: Сила = `8`. Только `[8,8]` эффективна — после удаления OR = `0` < `8` ✓.

**Пример 4:**
- Ввод: `nums = [2,2,1]`
- Вывод: `5`
- Пояснение: Сила = `2 OR 2 OR 1 = 3`. Эффективные подпоследовательности:
  - `[1]`: остаток `[2,2]` → OR = `2` < `3` ✓
  - `[2,1]` (`nums[0], nums[2]`): остаток `[2]` → OR = `2` < `3` ✓
  - `[2,1]` (`nums[1], nums[2]`): остаток `[2]` → OR = `2` < `3` ✓
  - `[2,2]`: остаток `[1]` → OR = `1` < `3` ✓
  - `[2,2,1]`: остаток `[]` → OR = `0` < `3` ✓

**Ограничения:**
- `1 <= nums.length <= 10^5`
- `1 <= nums[i] <= 10^6`
